# Briefs

Implementation briefs live here.

Briefs translate accepted ADRs and plans into bounded execution sessions. A brief is an execution contract, not a strategy document.

Every Codex implementation brief must use this structure:

```text
Goal
Context
Non-goals
Files
Constraints
Implementation
Tests
Acceptance criteria
Rollback
Done definition
```

## Protocol

- Codex may propose an implementation plan inside an approved brief.
- Codex does not approve its own scope expansion.
- Any scope expansion goes back to the Chief Architect or Oleg.
- No implementation starts without a clear brief or explicit founder instruction.
